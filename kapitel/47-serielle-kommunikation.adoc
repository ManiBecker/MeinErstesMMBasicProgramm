== Serielle Kommunikation

image::bilder/47_serielle_kommunikation.png[align=center]

=== Das lernst du

In diesem Kapitel lernst du:

* was eine serielle Schnittstelle ist
* Daten über eine UART-Verbindung zu senden und zu empfangen
* einen PicoMite mit einem PC zu verbinden
* zwei MMBasic-Systeme miteinander kommunizieren zu lassen
* einfache Kommunikationsprotokolle zu entwickeln

Am Ende des Kapitels kannst du eigene Geräte miteinander Daten austauschen lassen.

=== Eine der ältesten Computerschnittstellen

Die serielle Datenübertragung ist deutlich älter als der Personal Computer. Bereits Anfang des 20. Jahrhunderts wurde sie bei Fernschreibern eingesetzt. Das heute noch gebräuchliche Start-/Stopbit-Verfahren entstand bereits in den 1920er-Jahren.

Mit dem Aufkommen der Mikrocomputer setzte sich in den 1970er- und 1980er-Jahren vor allem die RS-232-Schnittstelle als Standard für die serielle Kommunikation zwischen Computern und Peripheriegeräten durch.

Viele bekannte Systeme nutzten diese Technik:

* Commodore C64
* Atari ST
* Apple II
* CP/M-Rechner
* Robotron-Computer
* Industriecomputer
* Terminals

Damals war die serielle Schnittstelle oft die wichtigste Verbindung zur Außenwelt.

Auch heute wird sie noch in unzähligen Geräten eingesetzt.

=== Was bedeutet seriell?

Daten bestehen aus einzelnen Bits.

Bei einer seriellen Verbindung werden diese Bits nacheinander übertragen.

Beispiel:

[source]
----
H = 01001000
----

Die einzelnen Bits werden nacheinander über eine Leitung gesendet:

[source]
----
0 1 0 0 1 0 0 0
----

Daher stammt die Bezeichnung *seriell*.

=== UART

Die meisten Mikrocontroller besitzen eine UART.

UART steht für:

[source]
----
Universal Asynchronous
Receiver Transmitter
----

Die UART übernimmt das Senden und Empfangen der Daten.

Der Programmierer muss sich dadurch nicht um die einzelnen Bits kümmern.

=== Wo begegnet uns UART heute?

Serielle Schnittstellen werden heute beispielsweise verwendet für:

* GPS-Empfänger
* Sensoren
* Funkmodule
* industrielle Steuerungen
* Retrocomputer
* andere Mikrocontroller

Sie gehören zu den wichtigsten Kommunikationsschnittstellen in der Elektronik.

=== Baudrate

Damit zwei Geräte miteinander kommunizieren können, müssen beide dieselbe Geschwindigkeit verwenden.

Diese Geschwindigkeit wird als Baudrate bezeichnet.

Typische Werte sind:

[source]
----
1200
2400
4800
9600
19200
38400
115200
----

Für viele Anwendungen sind 9600 oder 115200 Baud üblich.

=== Sender und Empfänger

Eine UART besitzt zwei Signale:

[cols="1,3"]
|===
|Signal |Bedeutung

|TX
|Senden

|RX
|Empfangen
|===

Wichtig:

[source]
----
TX → RX

RX → TX
----

Sender und Empfänger werden gekreuzt verbunden.

=== Verbindung zweier Systeme

[source]
----
PicoMite A          PicoMite B

TX ---------------- RX

RX ---------------- TX

GND -------------- GND
----

Zusätzlich müssen beide Geräte eine gemeinsame Masse besitzen.

=== Die erste Ausgabe

MMBasic behandelt eine serielle Schnittstelle ähnlich wie eine Datei.

Zunächst wird die Schnittstelle geöffnet.

Anschließend können Daten geschrieben oder gelesen werden.

=== Daten senden

Beispiel:

[source,basic]
----
OPEN "COM1:9600" AS #1

PRINT #1, "Hallo Welt"
----

Der Text wird über die serielle Schnittstelle gesendet.

=== Schnittstelle schließen

Nach der Verwendung sollte die Verbindung geschlossen werden.

[source,basic]
----
CLOSE #1
----

=== Daten empfangen

Zum Empfangen verwenden wir:

[source,basic]
----
LINE INPUT #1, text$

PRINT text$
----

Die empfangene Zeile wird in einer Variablen gespeichert.

=== Praxisprojekt: PicoMite und PC verbinden

Ein besonders nützliches Experiment ist die Verbindung mit einem PC.

=== Benötigte Software

Geeignete Terminalprogramme sind beispielsweise:

* Tera Term
* PuTTY
* MobaXterm

Diese Programme können serielle Daten anzeigen und senden.

=== Mehr als nur Text: VT100-Terminals

Viele Terminalprogramme unterstützen den sogenannten *VT100-Standard* bzw. ANSI-Steuersequenzen. Ursprünglich wurde dieser Standard Ende der 1970er-Jahre für Computerterminals von Digital Equipment Corporation (DEC) entwickelt.

Dadurch kann ein Programm nicht nur Text ausgeben, sondern auch den Bildschirm löschen, den Cursor an eine bestimmte Position setzen oder farbigen Text anzeigen.

MMBasic kann diese Steuersequenzen wie gewöhnlichen Text über die serielle Schnittstelle senden. Dadurch lassen sich komfortable textbasierte Benutzeroberflächen erstellen – ganz ohne grafisches Display.

Zum Beispiel:

[source,basic]
----
PRINT #1, CHR$(27) + "[2J"    ' Bildschirm löschen
PRINT #1, CHR$(27) + "[H"     ' Cursor nach links oben
PRINT #1, "Hallo VT100!"
----

TIP: Im Anhang findest du eine Übersicht der wichtigsten ASCII-Zeichen sowie der gebräuchlichsten VT100-/ANSI-Steuersequenzen.

=== Testprogramm

[source,basic]
----
OPEN "COM1:9600" AS #1

PRINT #1, "Hallo vom PicoMite"

CLOSE #1
----

Wird nun ein Terminalprogramm gestartet, erscheint die gesendete Nachricht auf dem Bildschirm.

=== Praxisprojekt: Zwei PicoMites verbinden

Nun verbinden wir zwei MMBasic-Systeme miteinander.

=== Sender

[source,basic]
----
OPEN "COM1:9600" AS #1

DO

  PRINT #1, "Hallo"

  PAUSE 1000

LOOP
----

=== Empfänger

[source,basic]
----
OPEN "COM1:9600" AS #1

DO

  LINE INPUT #1, text$

  PRINT text$

LOOP
----

Der Empfänger zeigt nun jede Sekunde:

[source]
----
Hallo
----

an.

=== Ein einfacher Chat

Interessanter wird es, wenn beide Seiten senden und empfangen können.

Damit entsteht ein einfacher Chat.

=== Sender

[source,basic]
----
OPEN "COM1:9600" AS #1

DO

  INPUT "Text"; text$

  PRINT #1, text$

LOOP
----

=== Empfänger

[source,basic]
----
OPEN "COM1:9600" AS #1

DO

  LINE INPUT #1, text$

  PRINT text$

LOOP
----

Nun können Nachrichten zwischen zwei Geräten ausgetauscht werden.

=== Nachrichten strukturieren

Bei größeren Projekten empfiehlt es sich, feste Nachrichtenformate zu verwenden.

Beispiel:

[source]
----
TEMP:23.5

LED:ON

LED:OFF
----

Das empfangende Programm kann diese Nachrichten gezielt auswerten.

=== Praxisprojekt: Fernsteuerung einer LED

Sender:

[source,basic]
----
OPEN "COM1:9600" AS #1

PRINT #1, "LEDON"
----

Empfänger:

[source,basic]
----
OPEN "COM1:9600" AS #1

SETPIN GP15, DOUT

DO

  LINE INPUT #1, text$

  IF text$ = "LEDON" THEN
    PIN(GP15)=1
  ENDIF

LOOP
----

Damit kann ein Mikrocontroller einen anderen fernsteuern.

=== Retrocomputer und serielle Kommunikation

Viele klassische Computer verfügen über serielle Schnittstellen.

Beispiele:

* AC1
* Z1013
* C64 mit RS232-Adapter
* Apple II
* CP/M-Rechner
* Siemens PGs
* Eigenbaucomputer

Ein PicoMite kann dadurch als moderne Erweiterung dienen.

=== Praktische Anwendungen

Beispielsweise:

* Programme übertragen
* Messdaten speichern
* SD-Karten-Zugriff ermöglichen
* Terminalfunktionen bereitstellen
* Sensorwerte anzeigen

Dadurch lassen sich moderne und historische Systeme miteinander verbinden.

=== Praxisprojekt: Zeitmessung für einen Commodore CBM

In einem Retrocomputer-Forum stellte ein Leser die folgende Frage:

[quote]
____
Ich brauche demnächst die Anbindung eines Pi Pico an einen Commodore CBM zur Zeiterfassung von RC-Fahrzeugen. Die Zeit soll mit einer Lichtschranke gemessen werden.

Der CBM ist dafür eigentlich zu langsam, der Pi Pico kann dagegen problemlos Hundertstelsekunden oder sogar Millisekunden erfassen.

Die Daten sollen anschließend über eine serielle Verbindung an den CBM übertragen werden.
____

Dieses Beispiel zeigt sehr schön, wie sich moderne Mikrocontroller und klassische Computer sinnvoll ergänzen können.

Der PicoMite übernimmt die Aufgabe, die hohe Rechengeschwindigkeit und präzise Zeitmessung erfordert, während der Commodore CBM die Ergebnisse anzeigt, speichert oder weiterverarbeitet.

=== Aufgaben sinnvoll verteilen

Beide Computer besitzen unterschiedliche Stärken.

Der Commodore CBM bietet:

* komfortable BASIC-Programmierung
* Tastatur und Bildschirm
* einfache Datenauswertung
* gute Möglichkeiten zur Speicherung der Ergebnisse

Der PicoMite bietet dagegen:

* einen schnellen Mikrocontroller
* präzise Timer
* Interrupts
* schnelle Reaktion auf Eingangssignale

Anstatt den CBM durch einen modernen Rechner zu ersetzen, übernimmt der PicoMite lediglich die Aufgabe, für die er besser geeignet ist.

=== Aufbau der Schaltung

Der prinzipielle Aufbau sieht folgendermaßen aus:

[source]
----
             Lichtschranke
                    │
                    ▼
           +----------------+
           |    PicoMite    |
           |                |
           | Zeitmessung    |
           | Interrupt      |
           | TIMER          |
           +----------------+
                    │
             UART / RS232
                    │
                    ▼
         +--------------------+
         | Commodore CBM      |
         | BASIC-Programm     |
         | Anzeige            |
         | Auswertung         |
         +--------------------+
----

Sobald das Fahrzeug die Lichtschranke durchfährt, registriert der PicoMite das Ereignis und berechnet die verstrichene Zeit.

Anschließend wird das Ergebnis über die serielle Schnittstelle an den Commodore übertragen.

=== Warum übernimmt der CBM die Zeitmessung nicht selbst?

Natürlich könnte man versuchen, die Zeit direkt im BASIC-Programm des CBM zu messen.

Allerdings arbeitet BASIC interpretiert. Während der Rechner gleichzeitig Tastatur, Bildschirm und Programm ausführt, vergeht bereits wertvolle Zeit.

Für viele Anwendungen reicht diese Genauigkeit völlig aus.

Bei einer Rennstrecke oder einer Lichtschranke möchte man jedoch möglichst genaue Messwerte erhalten.

Ein Mikrocontroller wie der PicoMite reagiert innerhalb weniger Mikrosekunden auf ein Eingangssignal und kann Zeitabstände sehr präzise bestimmen.

=== Die serielle Verbindung

Der RP2040 bzw. RP2350 besitzt bereits eine integrierte UART.

Für die serielle Kommunikation ist daher keine zusätzliche Hardware erforderlich.

Soll die Verbindung jedoch an eine klassische RS232-Schnittstelle erfolgen, müssen die unterschiedlichen Signalpegel angepasst werden.

Hierzu verwendet man einen Pegelwandler wie beispielsweise den bekannten MAX232.

[source]
----
PicoMite (3,3 V UART)
          │
          ▼
      MAX232
          │
          ▼
RS232 des Commodore
----

Besitzt der Zielrechner dagegen bereits TTL-Pegel oder wird der User-Port entsprechend angepasst, kann auf den MAX232 verzichtet werden.

=== Ein einfaches Kommunikationsprotokoll

Der PicoMite muss dem Commodore lediglich die gemessene Zeit übertragen.

Eine einfache Textnachricht genügt bereits.

Beispiele:

[source]
----
TIME:2.37
----

oder

[source]
----
TIME:2371
----

Im zweiten Beispiel wird die Zeit in Millisekunden übertragen.

Textbasierte Protokolle besitzen den Vorteil, dass sie leicht lesbar und einfach zu testen sind.

=== Das BASIC-Programm auf dem Commodore

Der Commodore muss die empfangene Zeichenkette lediglich einlesen.

Das Programm könnte anschließend beispielsweise folgende Ausgabe erzeugen:

[source]
----
Runde 1 : 2.371 s
Runde 2 : 2.294 s
Runde 3 : 2.415 s
----

Die eigentliche Berechnung der Zeit erfolgt vollständig auf dem PicoMite.

=== Erweiterungsmöglichkeiten

Aus dieser einfachen Idee lassen sich viele interessante Projekte entwickeln.

Zum Beispiel:

* Rundenzähler
* Bestzeiten speichern
* Mehrere Lichtschranken auswerten
* Fehlstarts erkennen
* Ziel- und Zwischenzeiten messen
* Ergebnisse auf SD-Karte speichern
* Live-Anzeige auf einem zweiten Display

=== Mehr als nur ein Commodore

Das gleiche Prinzip funktioniert selbstverständlich auch mit vielen anderen klassischen Computern.

Zum Beispiel:

* AC1
* Z1013
* KC85
* Apple II
* CP/M-Rechner
* Siemens PG
* Eigenbaucomputer mit serieller Schnittstelle

Der Retrocomputer übernimmt dabei die Benutzeroberfläche, während der PicoMite die zeitkritischen Aufgaben erledigt.

[TIP]
====
Viele Retrocomputer besitzen bereits eine serielle Schnittstelle oder können mit einem einfachen RS232-Interface nachgerüstet werden.

Dadurch lassen sich moderne Sensoren, Mikrocontroller und Messgeräte überraschend einfach mit historischen Computern verbinden.
====

=== Fazit

Dieses Projekt zeigt eindrucksvoll, dass ein PicoMite keinen klassischen Computer ersetzen muss.

Im Gegenteil: Beide Systeme ergänzen sich hervorragend.

Der Retrocomputer übernimmt die Bedienung und Darstellung der Daten, während der PicoMite seine hohe Geschwindigkeit für präzise Messungen nutzt.

Genau diese Kombination macht den besonderen Reiz vieler moderner Retro-Projekte aus.

=== Ein Blick in die Zukunft

Serielle Kommunikation bildet die Grundlage vieler weiterer Technologien.

Dazu gehören:

* Bluetooth-Module
* WLAN-Module
* GPS-Empfänger
* Funkmodule
* industrielle Feldbusse

Wer UART versteht, versteht auch viele andere Kommunikationssysteme deutlich schneller.

=== Tipps aus der Praxis

* Sender und Empfänger müssen dieselbe Baudrate verwenden.
* TX und RX werden gekreuzt verbunden.
* Vergiss die gemeinsame Masse nicht.
* Beginne mit 9600 Baud.
* Verwende einfache Textprotokolle.
