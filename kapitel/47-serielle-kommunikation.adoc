== Serielle Kommunikation

=== Das lernst du

In diesem Kapitel lernst du:

* was eine serielle Schnittstelle ist
* Daten über eine UART-Verbindung zu senden und zu empfangen
* einen PicoMite mit einem PC zu verbinden
* zwei MMBasic-Systeme miteinander kommunizieren zu lassen
* einfache Kommunikationsprotokolle zu entwickeln

Am Ende des Kapitels kannst du eigene Geräte miteinander Daten austauschen lassen.

=== Eine der ältesten Computerschnittstellen

Bereits in den 1970er- und 1980er-Jahren kommunizierten Computer über serielle Schnittstellen miteinander.

Viele bekannte Systeme nutzten diese Technik:

* Commodore C64
* Atari ST
* Apple II
* CP/M-Rechner
* Robotron-Computer
* Industriecomputer
* Terminals

Damals war die serielle Schnittstelle oft die wichtigste Verbindung zur Außenwelt.

Auch heute wird sie noch in unzähligen Geräten eingesetzt.

=== Was bedeutet seriell?

Daten bestehen aus einzelnen Bits.

Bei einer seriellen Verbindung werden diese Bits nacheinander übertragen.

Beispiel:

[source]
----
H = 01001000
----

Die einzelnen Bits werden nacheinander über eine Leitung gesendet:

[source]
----
0 1 0 0 1 0 0 0
----

Daher stammt die Bezeichnung *seriell*.

=== UART

Die meisten Mikrocontroller besitzen eine UART.

UART steht für:

[source]
----
Universal Asynchronous
Receiver Transmitter
----

Die UART übernimmt das Senden und Empfangen der Daten.

Der Programmierer muss sich dadurch nicht um die einzelnen Bits kümmern.

=== Wo begegnet uns UART heute?

Serielle Schnittstellen werden heute beispielsweise verwendet für:

* GPS-Empfänger
* Sensoren
* Funkmodule
* industrielle Steuerungen
* Retrocomputer
* andere Mikrocontroller

Sie gehören zu den wichtigsten Kommunikationsschnittstellen in der Elektronik.

=== Baudrate

Damit zwei Geräte miteinander kommunizieren können, müssen beide dieselbe Geschwindigkeit verwenden.

Diese Geschwindigkeit wird als Baudrate bezeichnet.

Typische Werte sind:

[source]
----
1200
2400
4800
9600
19200
38400
115200
----

Für viele Anwendungen sind 9600 oder 115200 Baud üblich.

=== Sender und Empfänger

Eine UART besitzt zwei Signale:

[cols="1,3"]
|===
|Signal |Bedeutung

|TX
|Senden

|RX
|Empfangen
|===

Wichtig:

[source]
----
TX → RX

RX → TX
----

Sender und Empfänger werden gekreuzt verbunden.

=== Verbindung zweier Systeme

[source]
----
PicoMite A          PicoMite B

TX ---------------- RX

RX ---------------- TX

GND -------------- GND
----

Zusätzlich müssen beide Geräte eine gemeinsame Masse besitzen.

=== Die erste Ausgabe

MMBasic behandelt eine serielle Schnittstelle ähnlich wie eine Datei.

Zunächst wird die Schnittstelle geöffnet.

Anschließend können Daten geschrieben oder gelesen werden.

=== Daten senden

Beispiel:

[source,basic]
----
OPEN "COM1:9600" AS #1

PRINT #1, "Hallo Welt"
----

Der Text wird über die serielle Schnittstelle gesendet.

=== Schnittstelle schließen

Nach der Verwendung sollte die Verbindung geschlossen werden.

[source,basic]
----
CLOSE #1
----

=== Daten empfangen

Zum Empfangen verwenden wir:

[source,basic]
----
LINE INPUT #1, text$

PRINT text$
----

Die empfangene Zeile wird in einer Variablen gespeichert.

=== Praxisprojekt: PicoMite und PC verbinden

Ein besonders nützliches Experiment ist die Verbindung mit einem PC.

=== Benötigte Software

Geeignete Terminalprogramme sind beispielsweise:

* Tera Term
* PuTTY
* MobaXterm

Diese Programme können serielle Daten anzeigen und senden.

=== Testprogramm

[source,basic]
----
OPEN "COM1:9600" AS #1

PRINT #1, "Hallo vom PicoMite"

CLOSE #1
----

Wird nun ein Terminalprogramm gestartet, erscheint die gesendete Nachricht auf dem Bildschirm.

=== Praxisprojekt: Zwei PicoMites verbinden

Nun verbinden wir zwei MMBasic-Systeme miteinander.

=== Sender

[source,basic]
----
OPEN "COM1:9600" AS #1

DO

  PRINT #1, "Hallo"

  PAUSE 1000

LOOP
----

=== Empfänger

[source,basic]
----
OPEN "COM1:9600" AS #1

DO

  LINE INPUT #1, text$

  PRINT text$

LOOP
----

Der Empfänger zeigt nun jede Sekunde:

[source]
----
Hallo
----

an.

=== Ein einfacher Chat

Interessanter wird es, wenn beide Seiten senden und empfangen können.

Damit entsteht ein einfacher Chat.

=== Sender

[source,basic]
----
OPEN "COM1:9600" AS #1

DO

  INPUT "Text"; text$

  PRINT #1, text$

LOOP
----

=== Empfänger

[source,basic]
----
OPEN "COM1:9600" AS #1

DO

  LINE INPUT #1, text$

  PRINT text$

LOOP
----

Nun können Nachrichten zwischen zwei Geräten ausgetauscht werden.

=== Nachrichten strukturieren

Bei größeren Projekten empfiehlt es sich, feste Nachrichtenformate zu verwenden.

Beispiel:

[source]
----
TEMP:23.5

LED:ON

LED:OFF
----

Das empfangende Programm kann diese Nachrichten gezielt auswerten.

=== Praxisprojekt: Fernsteuerung einer LED

Sender:

[source,basic]
----
OPEN "COM1:9600" AS #1

PRINT #1, "LEDON"
----

Empfänger:

[source,basic]
----
OPEN "COM1:9600" AS #1

SETPIN GP15, DOUT

DO

  LINE INPUT #1, text$

  IF text$ = "LEDON" THEN
    PIN(GP15)=1
  ENDIF

LOOP
----

Damit kann ein Mikrocontroller einen anderen fernsteuern.

=== Retrocomputer und serielle Kommunikation

Viele klassische Computer verfügen über serielle Schnittstellen.

Beispiele:

* AC1
* Z1013
* C64 mit RS232-Adapter
* Apple II
* CP/M-Rechner
* Siemens PGs
* Eigenbaucomputer

Ein PicoMite kann dadurch als moderne Erweiterung dienen.

=== Praktische Anwendungen

Beispielsweise:

* Programme übertragen
* Messdaten speichern
* SD-Karten-Zugriff ermöglichen
* Terminalfunktionen bereitstellen
* Sensorwerte anzeigen

Dadurch lassen sich moderne und historische Systeme miteinander verbinden.

=== Ein Blick in die Zukunft

Serielle Kommunikation bildet die Grundlage vieler weiterer Technologien.

Dazu gehören:

* Bluetooth-Module
* WLAN-Module
* GPS-Empfänger
* Funkmodule
* industrielle Feldbusse

Wer UART versteht, versteht auch viele andere Kommunikationssysteme deutlich schneller.

=== Tipps aus der Praxis

* Sender und Empfänger müssen dieselbe Baudrate verwenden.
* TX und RX werden gekreuzt verbunden.
* Vergiss die gemeinsame Masse nicht.
* Beginne mit 9600 Baud.
* Verwende einfache Textprotokolle.
