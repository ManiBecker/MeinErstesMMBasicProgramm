== Entscheidungen mit IF

image::bilder/05_entscheidungen_mit_if.png[align=center]

=== Das lernst du

In diesem Kapitel lernst du:

* den integrierten Editor von MMBasic kennen
* Programme zu speichern und auszuführen
* Entscheidungen mit `IF` zu treffen
* `ELSE` und `ELSEIF` zu verwenden
* Bedingungen mit Vergleichsoperatoren zu formulieren

Am Ende des Kapitels kannst du Programme schreiben, die auf unterschiedliche Situationen reagieren.

=== Programme werden größer

Bisher haben wir alle Beispiele direkt an der MMBasic-Eingabeaufforderung eingegeben.

Das war ideal, um einzelne Befehle auszuprobieren und ihre Wirkung sofort zu sehen.

Ab diesem Kapitel werden unsere Programme jedoch umfangreicher. Sie bestehen aus mehreren zusammengehörenden Zeilen und enthalten Programmstrukturen wie `IF`, `FOR` oder `DO...LOOP`.

Solche Programme schreibt man am einfachsten im integrierten Editor von MMBasic.

Keine Sorge – der Editor ist bewusst einfach gehalten und schnell zu erlernen.

=== Den Editor starten

Den Editor startest du mit dem Befehl:

[source,basic]
----
EDIT
----

Nun kannst du dein Programm Zeile für Zeile eingeben.

Während der Eingabe kannst du den Cursor mit den Pfeiltasten bewegen und den Text jederzeit ändern.

[TIP]
====
Falls sich bereits ein anderes Programm im Speicher befindet, kannst du es vorher mit

[source,basic]
----
NEW
----

löschen.

MMBasic fragt anschließend nach, ob das Programm wirklich gelöscht werden soll.
====

=== Programme speichern

Nachdem du dein Programm eingegeben hast, speicherst du es mit **F1**.

Das Programm bleibt auch nach dem Ausschalten des PicoMite erhalten.

Mit **F2** verlässt du anschließend den Editor und kehrst zur MMBasic-Eingabeaufforderung zurück.

Später in diesem Buch lernst du außerdem, wie Programme als Dateien im Dateisystem oder auf einer SD-Karte gespeichert und wieder geladen werden können.

=== Programme starten

Ein im Speicher befindliches Programm wird mit dem Befehl

[source,basic]
----
RUN
----

gestartet.

Möchtest du das Programm später ändern, öffnest du den Editor einfach erneut mit

[source,basic]
----
EDIT
----

=== Warum Entscheidungen wichtig sind

Bis jetzt wurden alle Programmzeilen nacheinander ausgeführt.

Oft soll ein Programm jedoch unterschiedlich reagieren.

Zum Beispiel:

* Ist eine Zahl größer als 100?
* Wurde ein Taster gedrückt?
* Ist eine Datei vorhanden?
* Hat der Benutzer die richtige Antwort eingegeben?

Für solche Entscheidungen gibt es den `IF`-Befehl.

=== Unsere erste Entscheidung

Das folgende Programm überprüft, ob die Variable `A` den Wert 10 besitzt.

[source,basic]
----
A = 10

IF A = 10 THEN
  PRINT "A hat den Wert 10."
ENDIF
----

Die Bedingung hinter `IF` wird geprüft.

Ist sie wahr, werden die Anweisungen zwischen `THEN` und `ENDIF` ausgeführt.

Ist sie falsch, werden diese Zeilen übersprungen.

=== Was bedeutet IF?

`IF` bedeutet auf Deutsch **wenn**.

Die allgemeine Schreibweise lautet:

[source,basic]
----
IF Bedingung THEN

  Anweisungen

ENDIF
----

Das Programm kann dadurch unterschiedliche Wege einschlagen.

=== ELSE – der andere Fall

Mit `ELSE` kann festgelegt werden, was passieren soll, wenn die Bedingung nicht erfüllt ist.

[source,basic]
----
A = 5

IF A = 10 THEN
  PRINT "A ist 10."
ELSE
  PRINT "A ist nicht 10."
ENDIF
----

Da `A` den Wert 5 besitzt, erscheint:

[source]
----
A ist nicht 10.
----

=== Vergleichsoperatoren

Für Bedingungen werden Vergleichsoperatoren verwendet.

[cols="1,3",options="header"]
|===
|Operator
|Bedeutung

|=
|gleich

|<>
|ungleich

|<
|kleiner

|>
|größer

|<=
|kleiner oder gleich

|>=
|größer oder gleich
|===

=== Unser erstes Quiz

Mit `IF` können Programme auf Benutzereingaben reagieren.

Das folgende Programm stellt eine einfache Rechenaufgabe.

[source,basic]
----
PRINT "Wieviel ist 2+3?"
INPUT ANTWORT

IF ANTWORT = 5 THEN
  PRINT "Richtig!"
ELSE
  PRINT "Leider falsch."
ENDIF
----

Beispiel:

[source]
----
Wieviel ist 2+3?
? 5

Richtig!
----

Obwohl das Programm nur aus wenigen Zeilen besteht, trifft es bereits eine Entscheidung.

Genau dieses Prinzip wird später in Spielen, Menüs oder Messprogrammen verwendet.

=== Mehrere Entscheidungen

Ein Programm kann beliebig viele Entscheidungen enthalten.

Im folgenden Beispiel wird für jede Schulnote eine eigene `IF`-Anweisung verwendet.

[source,basic]
----
INPUT NOTE

IF NOTE = 1 THEN
  PRINT "Sehr gut"
ENDIF

IF NOTE = 2 THEN
  PRINT "Gut"
ENDIF

IF NOTE = 3 THEN
  PRINT "Befriedigend"
ENDIF
----

Das funktioniert zwar, ist bei vielen Möglichkeiten jedoch etwas umständlich.

=== ELSEIF verwenden

Mehrere Möglichkeiten lassen sich deutlich übersichtlicher mit `ELSEIF` formulieren.

[source,basic]
----
INPUT NOTE

IF NOTE = 1 THEN
  PRINT "Sehr gut"
ELSEIF NOTE = 2 THEN
  PRINT "Gut"
ELSEIF NOTE = 3 THEN
  PRINT "Befriedigend"
ELSE
  PRINT "Unbekannte Note"
ENDIF
----

Beispiel:

[source]
----
? 2

Gut
----

Sobald eine Bedingung erfüllt ist, werden die restlichen `ELSEIF`-Bedingungen nicht mehr geprüft.

Programme werden dadurch übersichtlicher und leichter zu erweitern.

=== Verschachtelte Entscheidungen

Auch innerhalb eines `IF`-Blocks kann sich eine weitere Entscheidung befinden.

[source,basic]
----
ALTER = 20

IF ALTER >= 18 THEN

  PRINT "Volljährig"

  IF ALTER >= 67 THEN
    PRINT "Rentenalter erreicht"
  ENDIF

ENDIF
----

Solche verschachtelten Entscheidungen kommen in größeren Programmen häufig vor.

=== Häufige Fehler

Beim Arbeiten mit `IF` passieren Anfängern häufig dieselben Fehler.

Achte besonders auf folgende Punkte:

* Nach der Bedingung muss `THEN` stehen.
* Jeder `IF`-Block benötigt ein passendes `ENDIF`.
* Für Vergleiche wird `=` verwendet.
* Die Anweisungen zwischen `THEN` und `ENDIF` sollten eingerückt werden. Dadurch werden Programme wesentlich übersichtlicher.

=== Probier's selbst!

Versuche folgende Aufgaben:

. Schreibe ein Programm, das prüft, ob eine Zahl größer als 100 ist.
. Gib aus, ob eine Zahl gerade oder ungerade ist.
. Vergleiche zwei Zahlen miteinander.
. Erweitere das Quiz um weitere Rechenaufgaben.
. Gib abhängig vom Alter unterschiedliche Texte aus.

In diesem Kapitel hast du gelernt,

* den Editor von MMBasic zu verwenden,

* Entscheidungen mit `IF` zu treffen,
* `ELSE` und `ELSEIF` einzusetzen,
* Bedingungen mit Vergleichsoperatoren zu formulieren,
