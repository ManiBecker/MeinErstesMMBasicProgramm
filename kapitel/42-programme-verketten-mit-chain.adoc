== Programme verketten mit CHAIN

=== Das lernst du

In diesem Kapitel lernst du:

* was `CHAIN` macht,
* wann `CHAIN` sinnvoll eingesetzt wird,
* wie du mehrere Programme miteinander verbindest,
* wie du ein gemeinsames Menü für deine Programme erstellst und
* wie Daten zwischen getrennten Programmen ausgetauscht werden können.

Am Ende des Kapitels kannst du mehrere eigenständige MMBasic-Programme zu
einer gemeinsamen Anwendung verbinden.

=== Ein gemeinsames Menü

In den bisherigen Kapiteln hast du bereits mehrere Programme entwickelt,
zum Beispiel:

* Pong,
* Senso,
* Little Professor und
* die Bahnhofsuhr.

Wäre es nicht praktisch, wenn du alle diese Programme über ein gemeinsames
Hauptmenü starten könntest?

Genau dafür gibt es den Befehl `CHAIN`.

Mit `CHAIN` lassen sich mehrere eigenständige Programme miteinander verbinden.
Dadurch können übersichtliche Anwendungen entstehen, deren einzelne Programme
jeweils eine bestimmte Aufgabe übernehmen.

Ein solches Projekt könnte beispielsweise aus folgenden Dateien bestehen:

[source]
----
HAUPTMENU.BAS
PONG.BAS
SENSO.BAS
LITTLEPROFESSOR.BAS
UHR.BAS
RECHNER.BAS
----

=== Was macht CHAIN?

Mit `CHAIN` wird ein anderes BASIC-Programm geladen und sofort gestartet.

Beispiel:

[source,basic]
----
CHAIN "PONG.BAS"
----

Das aktuell laufende Programm wird beendet. Anschließend wird `PONG.BAS`
geladen und gestartet.

[NOTE]
====
`CHAIN` startet das neue Programm so, als wäre es gerade frisch geladen
worden. Variablen des vorherigen Programms stehen deshalb im neuen Programm
nicht mehr zur Verfügung.
====

=== CHAIN oder RUN?

Auf den ersten Blick scheint `CHAIN` dem Befehl `RUN` ähnlich zu sein. Es gibt
jedoch einen wichtigen Unterschied.

`RUN` wird normalerweise verwendet, um ein Programm an der
MMBasic-Eingabeaufforderung zu starten:

[source,basic]
----
RUN "PONG.BAS"
----

`CHAIN` wird dagegen innerhalb eines bereits laufenden Programms eingesetzt:

[source,basic]
----
IF taste$ = "P" THEN
  CHAIN "PONG.BAS"
ENDIF
----

Damit kann ein Programm gezielt ein anderes Programm aufrufen.

=== Ein einfaches Hauptmenü

Erstellen wir ein Menü, über das sich einige unserer bisherigen Programme
starten lassen.

Speichere das folgende Programm als `HAUPTMENU.BAS`:

[source,basic]
----
CLS

PRINT "========================"
PRINT "   MMBASIC-SAMMLUNG"
PRINT "========================"
PRINT
PRINT "1 - Pong"
PRINT "2 - Senso"
PRINT "3 - Little Professor"
PRINT "4 - Bahnhofsuhr"
PRINT "5 - Rechner"
PRINT
PRINT "0 - Ende"

INPUT "Auswahl"; wahl

SELECT CASE wahl

  CASE 1
    CHAIN "PONG.BAS"

  CASE 2
    CHAIN "SENSO.BAS"

  CASE 3
    CHAIN "LITTLEPROFESSOR.BAS"

  CASE 4
    CHAIN "UHR.BAS"

  CASE 5
    CHAIN "RECHNER.BAS"

  CASE 0
    END

  CASE ELSE
    PRINT "Ungültige Auswahl"

END SELECT
----

Wählt der Benutzer einen Menüpunkt aus, lädt `CHAIN` das zugehörige Programm
und startet es.

=== Die Struktur des Projekts

Die Dateien können gemeinsam in einem Projektverzeichnis abgelegt werden:

[source]
----
MMBASIC-SAMMLUNG
│
├── HAUPTMENU.BAS
├── PONG.BAS
├── SENSO.BAS
├── LITTLEPROFESSOR.BAS
├── UHR.BAS
└── RECHNER.BAS
----

Der Benutzer startet nur das Hauptmenü:

[source,basic]
----
RUN "HAUPTMENU.BAS"
----

Alle weiteren Programme werden anschließend über das Menü aufgerufen.

=== Zurück zum Hauptmenü

Damit eine zusammenhängende Anwendung entsteht, sollten die aufgerufenen
Programme wieder zum Hauptmenü zurückkehren können.

Am Ende eines Spiels könnte beispielsweise stehen:

[source,basic]
----
PRINT
PRINT "Beliebige Taste für das Hauptmenü ..."

DO
LOOP UNTIL INKEY$ <> ""

CHAIN "HAUPTMENU.BAS"
----

Nach dem Tastendruck wird das Hauptmenü erneut geladen und gestartet.

=== Vorteile von CHAIN

Die Verbindung mehrerer Programme bietet einige Vorteile:

* Die einzelnen Programme bleiben überschaubar.
* Jedes Programm kann unabhängig entwickelt und getestet werden.
* Fehler lassen sich leichter eingrenzen.
* Einzelne Programme können ausgetauscht oder erweitert werden.
* Ein zentrales Menü kann verschiedene Programme zu einer gemeinsamen
  Anwendung verbinden.

[TIP]
====
`CHAIN` eignet sich besonders für Programme, die weitgehend unabhängig
voneinander arbeiten.

Sollen dagegen mehrere Programme dieselben Unterprogramme oder Funktionen
verwenden, ist häufig die MMBasic-Library die bessere Wahl.
====

=== Ein Beispiel aus der Praxis

Auch eine kleine Werkzeugsammlung lässt sich mit `CHAIN` aufbauen:

[source]
----
TOOLS
│
├── MENU.BAS
├── TABELLE.BAS
├── UHR.BAS
├── RECHNER.BAS
└── NOTIZEN.BAS
----

Das Menü übernimmt nur die Auswahl. Die einzelnen Programme bleiben
unabhängig voneinander und können jeweils separat gestartet und getestet
werden.

=== Grenzen von CHAIN

`CHAIN` lädt das neue Programm vollständig und beendet das aktuell laufende
Programm.

Dadurch gehen alle Variablen des bisherigen Programms verloren.

Beispiel:

[source,basic]
----
punkte = 100

CHAIN "SPIEL2.BAS"
----

Im Programm `SPIEL2.BAS` existiert die Variable `punkte` nicht mehr.

`CHAIN` verbindet also den Ablauf mehrerer Programme, aber nicht deren
Variablen oder Programmspeicher.

=== Daten zwischen Programmen austauschen

Sollen Daten über einen Programmwechsel hinweg erhalten bleiben, müssen sie
vorher gespeichert werden.

Dafür gibt es verschiedene Möglichkeiten:

* Daten in einer Datei speichern,
* Einstellungen dauerhaft ablegen oder
* benötigte Werte im neuen Programm erneut berechnen.

Für viele Anwendungen genügt eine kleine Datei.

==== Beispiel: Spielstand speichern

Das erste Programm speichert den Punktestand:

[source,basic]
----
OPEN "PUNKTE.TXT" FOR OUTPUT AS #1

PRINT #1, 150

CLOSE #1

CHAIN "SPIEL2.BAS"
----

Das zweite Programm liest ihn wieder ein:

[source,basic]
----
OPEN "PUNKTE.TXT" FOR INPUT AS #1

INPUT #1, punkte

CLOSE #1

PRINT "Punkte:"; punkte
----

So können Informationen von einem Programm an das nächste übergeben werden.

=== CHAIN und Verzeichnisse

`CHAIN` kann auch mit vollständigen Pfadangaben verwendet werden.

Beispiel:

[source,basic]
----
CHAIN "/SPIELE/PONG.BAS"
----

oder:

[source,basic]
----
CHAIN "/TOOLS/UHR.BAS"
----

Dadurch lassen sich auch größere Programmsammlungen übersichtlich in
verschiedenen Verzeichnissen organisieren.

=== Historischer Hintergrund

Schon viele klassische BASIC-Systeme kannten Befehle, mit denen zwischen
mehreren Programmen gewechselt werden konnte.

Dadurch konnten auch auf Computern mit wenig Speicher umfangreichere
Anwendungen entstehen. MMBasic führt dieses bewährte Konzept auf modernen
Systemen fort.

=== Praxisprojekt: Deine MMBasic-Sammlung

Erstelle eine gemeinsame Programmsammlung aus einigen Programmen, die du in
den bisherigen Kapiteln entwickelt hast.

Eine mögliche Struktur ist:

[source]
----
MMBasic-Sammlung
│
├── Hauptmenü
├── Spiele
├── Lernprogramme
└── Werkzeuge
----

Das Hauptmenü könnte beispielsweise folgende Einträge enthalten:

[source]
----
1 - Pong
2 - Senso
3 - Little Professor
4 - Bahnhofsuhr
5 - Rechner
----

Jedes aufgerufene Programm sollte am Ende wieder mit

[source,basic]
----
CHAIN "HAUPTMENU.BAS"
----

zum Hauptmenü zurückkehren.

Je größer deine Sammlung wird, desto nützlicher wird ein zentrales Menü.

=== Tipps aus der Praxis

* Verwende `CHAIN` für Programme, die weitgehend unabhängig voneinander
  arbeiten.
* Teste jedes Programm zunächst einzeln.
* Verwende aussagekräftige Datei- und Verzeichnisnamen.
* Speichere wichtige Daten vor dem Programmwechsel in einer Datei.
* Sorge dafür, dass jedes Unterprogramm wieder zum Hauptmenü zurückkehren kann.
* Plane frühzeitig eine klare Verzeichnisstruktur.

=== Probier's selbst!

. Erstelle ein Hauptmenü.
. Lege zwei kleine Testprogramme an.
. Starte beide Programme über `CHAIN`.
. Baue in jedes Programm eine Rückkehr zum Hauptmenü ein.
. Erweitere das Menü um weitere Programme.
. Übertrage einen Wert mithilfe einer Datei von einem Programm zum nächsten.

=== Zusammenfassung

In diesem Kapitel hast du gelernt:

* Programme mit `CHAIN` zu starten,
* mehrere eigenständige Programme miteinander zu verbinden,
* ein gemeinsames Menü zu erstellen,
* jedes Programm unabhängig zu entwickeln und zu testen und
* Daten mithilfe einer Datei zwischen Programmen auszutauschen.

`CHAIN` verbindet keine Quelltextmodule, sondern startet nacheinander
vollständige Programme. Damit eignet sich der Befehl besonders für
Menüsysteme und Programmsammlungen, deren einzelne Bestandteile weitgehend
unabhängig voneinander arbeiten.
